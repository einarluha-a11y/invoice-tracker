---
description: How to correctly handle bug fixes, extraction errors, or missing data updates
---
Когда пользователь (или вы сами) находите ошибку записи или парсинга, вы обязаны действовать строго по следующему алгоритму:

1. **Находится ошибка**: Определение недостающих данных или сломанной логики (например, перепутанные файлы, удаление отрицательных сумм).
2. **Исправляется в системе причина ошибки**: Внесение фундаментальных изменений в код бэкенда (`index.js`, `dashboard_auditor_agent.cjs`, системные промпты Claude), чтобы эта проблема больше никогда не возникала ни для каких будущих файлов.
3. **Обновление (Deploy)**: Сохранение системных исправлений в GitHub (`git commit` + `git push`), чтобы ядро системы было официально и перманентно обновлено.
4. **Прогонка файла через систему**: Написание временного скрипта для извлечения проблемного файла (из IMAP или Firebase Storage) и обязательный прогон этого файла **СТРОГО через обновленный системный пайплайн** (через `processInvoiceWithDocAI` и `auditAndProcessInvoice`). Никаких ручных `docRef.update()`!
5. **Ошибка записи исправляется**: Обновленная нейросеть возвращает идеальные метаданные, скрипт записывает их в Firestore Live-базу, после чего временный зонд удаляется.
