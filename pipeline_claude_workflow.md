# 🤝 CLAUDE ↔ PERPLEXITY PIPELINE PROTOCOL

## Кто что делает
- **Einar** — даёт задачу тебе (Claude) в десктоп-приложении
- **Claude (ты)** — анализируешь, предлагаешь решение, кодируешь, деплоишь
- **Perplexity** — независимый ревьюер: читает твои решения из GitHub, пишет замечания, одобряет или возвращает на доработку

## Канал связи
Все договорённости — через файлы в `_agents/pipeline/`:
- `SOLUTION.md` — **ты пишешь** сюда решение/код
- `REVIEW.md` — **Perplexity пишет** сюда ревью
- `STATUS.md` — текущий статус (информационный)

---

## ФАЗА 1: АРХИТЕКТУРА

### Что ты делаешь:
1. Изучи задачу от Einar
2. Изучи Chief Accountant Charter (`_agents/workflows/chief_accountant.md`)
3. Придумай решение
4. Запиши в `_agents/pipeline/SOLUTION.md` в формате:

```markdown
# SOLUTION

PHASE: ARCHITECTURE
ROUND: 1
TASK: [одна строка — суть задачи]

## ARCHITECTURE

### Анализ задачи
[что именно нужно сделать]

### Варианты решения
[2-3 подхода с плюсами/минусами]

### Выбранное решение
[какой подход и почему]

### План реализации
[конкретные файлы и изменения]

### Учтённые протоколы Charter
[какие правила Charter затрагивает эта задача]

### Риски
[что может пойти не так]
```

5. Сделай `git add _agents/pipeline/SOLUTION.md && git commit -m "claude: architecture round 1" && git push`
6. **Жди** — Perplexity прочитает и запишет ревью в `REVIEW.md`

### Читаем REVIEW.md:
- Открой файл после появления коммита от Perplexity
- Если `ВЕРДИКТ: APPROVED` → переходи к Фазе 2
- Если `ВЕРДИКТ: CHANGES_NEEDED`:
  - Прочитай замечания в секции `ЗАМЕЧАНИЯ:`
  - Исправь решение
  - Обнови `SOLUTION.md` — увеличь `ROUND: N` на 1
  - Запиши в новую секцию `## REVISION NOTES` что именно изменил
  - Запиши и запуши
  - Жди следующего ревью

## ФАЗА 2: КОД

### Что ты делаешь:
1. Реализуй одобренное архитектурное решение
2. Соблюдай **все** протоколы Charter, особенно:
   - После изменений в `automation/` → `pm2 restart all`
   - Числа через `parseNumGlobal`, никогда `parseFloat` напрямую
   - Никогда не придумывай VAT/рег. номера из памяти
   - Email идемпотентность: `markSeen` только после успешной записи в Firestore
3. Проверь синтаксис каждого файла: `node --check <файл>`
4. Запиши результат в `_agents/pipeline/SOLUTION.md` — **замени весь файл**:

```markdown
# SOLUTION

PHASE: CODE
ROUND: 1
TASK: [та же строка что в архитектуре]

## ARCHITECTURE
[скопируй одобренное решение сюда]

## CODE

### Что реализовано
[описание что сделано]

### Тесты синтаксиса
[вывод node --check для каждого изменённого файла]

### PM2 статус
[вывод pm2 status после restart, если менял automation/]

## CHANGED_FILES
- `automation/accountant_agent.cjs`
- `automation/document_ai_service.cjs`
[список всех изменённых файлов]
```

5. Закоммить изменения: `git add -A && git commit -m "claude: implement [task]" && git push`
6. **Жди** ревью от Perplexity в `REVIEW.md`

### Читаем REVIEW.md:
- Если `ВЕРДИКТ: CODE_APPROVED` → переходи к Фазе 3
- Если `ВЕРДИКТ: CODE_CHANGES_NEEDED`:
  - Прочитай `ПРОБЛЕМЫ:` — конкретные баги и нарушения
  - Исправь всё перечисленное
  - Обнови `SOLUTION.md` — увеличь `ROUND` на 1
  - Добавь секцию `## FIXES ROUND N` с описанием исправлений
  - Закоммить и запушить
  - Жди следующего ревью

---

## ФАЗА 3: ДЕПЛОЙ И ПРОВЕРКА

### Что ты делаешь:
1. Обнови `SOLUTION.md` — смени `PHASE: DEPLOYING`
2. Запусти деплой: `npm run deploy` (или `railway up` если Railway)
3. Дождись результата CI/CD
4. Запусти проверку что система работает (прогони тестовый инвойс через pipeline)
5. Запиши финальный результат в `SOLUTION.md`:

```markdown
DEPLOY_STATUS: OK
DEPLOY_NOTES: [что было задеплоено, какие тесты прошли]
```

или

```markdown
DEPLOY_STATUS: FAILED
DEPLOY_ERROR: [текст ошибки]
```

6. Закоммить и запушить

---

## Важные правила

1. **Никогда не пропускай фазы** — сначала архитектура одобрена, потом код
2. **Всегда увеличивай ROUND** при обновлении — иначе Perplexity не заметит изменение
3. **Пуши после каждого обновления** SOLUTION.md — Perplexity читает из GitHub
4. **Не жди Einar-а** — он делегировал процесс нам двоим
5. **Если не знаешь как исправить замечание** — напиши в SOLUTION.md секцию `## QUESTION_TO_PERPLEXITY` и запуши — агент прочитает
